# Qué?
Esta página busca compartir algunas buenas prácticas para trabajar de manera colaborativa en el proyecto del TP, con el objetivo de mejorar la comunicación, la organización y la calidad del trabajo en equipo.

## Cómo?
La rama master es la rama principal del proyecto, donde se encuentra el código estable y listo para ser utilizado. Para mantener la calidad del código y evitar conflictos, es importante seguir estas buenas prácticas:
- Crear ramas para cada tarea o funcionalidad: En lugar de trabajar directamente en la rama master, es recomendable crear ramas separadas para cada tarea o funcionalidad que estés desarrollando. Esto te permite trabajar de manera independiente sin afectar el código principal y facilita la revisión y el merge de tus cambios.
	- Esto lo podés hacer de manera muy sencilla, desde Visual Studio mismo o desde la interfaz gráfica de GitHub Desktop, creando una nueva rama a partir de master y dándole un nombre descriptivo que refleje la tarea o funcionalidad que estás desarrollando. Imagen referencia (https://drive.google.com/file/d/1Xk9ZhmDAxsTSijIWDDUuAMDxTLwAM41L/view?usp=drive_link) New branch y le das un nombre descriptivo de la tarea a realizar para que te acuerdes.

Antes de hacer un commit al proyecto, asegurate primero de haber traído los cambios más actuales de la rama master a tu rama de trabajo. Esto se puede hacer fácilmente haciendo un pull desde master a tu rama, lo que te permitirá resolver cualquier conflicto que pueda surgir y asegurarte de que tu código esté actualizado con los últimos cambios realizados por otros miembros del equipo.

Al crear commits, es importante escribir mensajes claros y descriptivos que expliquen los cambios realizados. Esto facilita la revisión del código por parte de otros miembros del equipo y ayuda a mantener un historial de cambios comprensible.

## Para qué?
Este "acuerdo de convivencia" nos va a servir como equipo para mantener un flujo de trabajo organizado y eficiente, evitando conflictos y asegurando que el código en la rama master se mantenga estable y de alta calidad. Además, estas prácticas fomentan la colaboración y la comunicación entre los miembros del equipo, lo que es fundamental para el éxito del proyecto.
